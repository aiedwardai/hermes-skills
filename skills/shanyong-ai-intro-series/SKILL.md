# 善用AI 介绍系文章写作风格

## 触发条件

- 用户说"写一篇善用AI风格的文章"、"介绍系"、"教程类"
- 需要写工具介绍、产品上手、操作指南类文章
- 公众号目标为 **袋鼠工坊（善用AI）**

## 品牌定位

| 项目 | 内容 |
|------|------|
| 公众号 | 袋鼠工坊 |
| 作者 | 戴剑 |
| 栏目 | 善用AI |
| 定位 | AI First · AI Native · AI Inside |
| Slogan | AI First  AI Native  AI inside |

| 名言引用 | "经常庆功，就会成功。"——《繁花》 |
| 文章格式 | `善用AI \| 副标题` |

## 文章结构

### 开头部（顺序固定，不可调换）

```markdown
# 善用AI | [标题]

推荐标签

善用AI

Slogan
AI First  AI Native  AI inside

座右铭
限制AI能力的，不是AI，而是你的认知

名言引用
"经常庆功，就会成功。"——《繁花》

摘要
[一段话的摘要，概括全文核心价值]
```

### 正文风格

| 维度 | 特征 | 示例 |
|------|------|------|
| 句子节奏 | 短句为主，偶有长句蓄势 | "还真有。" / "试了一下，挺香。" |
| 语气 | 朋友分享，不是专家布道 | "写出来给你避避" |
| 个人叙事 | "我"字频繁出现 | "踩坑了"、"我试了几次" |
| 技术细节 | 具体命令、具体地址、完整步骤 | curl 命令、URL 全部给出 |
| 文化元素 | 轻松、接地气 | "挺香"、"避坑" |

### 开头模式

**推荐模式：场景痛点切入 → 疑问 → 简短肯定回答**

```
[场景痛点] 每天打开手机，各种新闻APP的红点，点不完……
[疑问] 有没有办法让AI帮我刷？
[肯定回答] 还真有。
```

**备选模式：直接抛结论 + 个人见证**

```
[一句话结论] 腾讯新闻出了一个龙虾技能，让AI每天帮你刷新闻。
[个人见证] 试了一下，挺香。
```

### 正文结构

```
[CUT到] 直接切入主题，不铺垫
[一句话判断] 亮点就三个
[1. 第一点] 一句话讲清楚，不展开
[2. 第二点] 同上
[3. 第三点] 同上
[踩坑经历] 踩坑了，写出来给你避避
[解决方法] 具体怎么做
[注意事项] 注意事项就一个
```

### 结尾结构

```markdown
[简短总结段落]

深度交流

公域交流止步于此，知识星球封闭交流。

对AI工具、智能体有研究兴趣的，后台私信进小群。
```

**结尾特点：**
- 不加「剑识」「剑曰」等收束词
- 自然总结，不刻意升华
- 提供交流入口（知识星球/私信进小群）

## 不合适的写法

| ❌ 不要写 | ✅ 要写 |
|-----------|--------|
| "功能列表说明书" | 一句话点出核心价值 |
| 官方话术写步骤 | 个人亲身经历做引子 |
| 深度哲学分析（那是剑胆琴新的定位） | 步骤清晰但不过度解释 |
| 事无巨细解释原理 | 踩坑、踩坑、踩坑 |

## 写作检查清单

发布前逐项检查：

- [ ] 标题格式：`善用AI | [副标题]`
- [ ] 开头部：推荐标签 → 善用AI → Slogan → 座右铭 → 名言 → 摘要（顺序正确）
- [ ] 开头：场景痛点切入，不是产品介绍
- [ ] 语气：朋友分享，不是专家/官方
- [ ] 个人叙事：有"我"、"踩坑"、"试了一下"
- [ ] 短句：每段不超过3-4行
- [ ] 具体步骤：命令/URL完整给出
- [ ] 结尾：不加剑识/剑曰，自然总结 + 交流入口
- [ ] 封面：善用AI品牌配色（column="善用AI", author="戴剑"）
